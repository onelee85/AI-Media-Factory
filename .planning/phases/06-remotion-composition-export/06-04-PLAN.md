<objective>
Wire the Python backend to Remotion's Node.js render pipeline via ComposeService and Celery task.

Purpose: This plan bridges the Python backend and the Remotion Node.js project. ComposeService validates that audio and subtitle files exist, constructs the render props JSON, invokes the Node.js render script via subprocess, and tracks the result in a Video database model. The Celery task makes this async so the API doesn't block during rendering.

Output: A complete Python integration where calling compose_video_task(audio_id, subtitle_id) asynchronously produces a rendered 1080p MP4 file.
</objective>

    ```python
        pass

    class ComposeService:
        def __init__(self, storage: StorageService | None = None):
            self.storage = storage or StorageService()
            self.remotion_script = Path("remotion/src/render.mjs")
    ```

    Methods:
    1. `build_render_props(audio_path: Path, subtitle_content: str, title: str = "", images: list[str] | None = None) -> dict`
       - Constructs the props dict for the Remotion render script
       - Resolves audio_path to absolute path
       - Returns: `{ "audioSrc": str, "subtitleText": str, "backgroundImages": list, "titleText": str }`

    2. `render(props: dict, output_path: Path) -> dict`
       - Writes props to a temp JSON file
       - Runs: `node remotion/src/render.mjs --props <temp.json> --output <output_path>`
       - Uses subprocess.run with timeout=600, capture_output=True
       - Returns: `{ "success": bool, "output_path": str, "stdout": str, "stderr": str }`
       - Raises ComposeServiceError on non-zero exit code

    3. `compose(audio_file_path: Path, subtitle_content: str, output_path: Path, title: str = "", images: list[str] | None = None) -> dict`
       - Validates audio file exists
       - Calls build_render_props()
       - Calls render()
       - Returns render result dict
