  <name>Task 2: Build SRT subtitle parser</name>
  <files>remotion/src/parseSubtitles.ts</files>
  <behavior>
    - parseSrt(srtText: string) returns SubtitleSegment[]
    - SubtitleSegment: { id: number, text: string, startFrame: number, endFrame: number, startMs: number, endMs: number }
    - Converts SRT timestamps (HH:MM:SS,mmm) to milliseconds and frame numbers
    - Handles multi-line subtitle text (joins with space)
    - Returns empty array for empty/invalid input
    - test: parseSrt("1\n00:00:01,000 --> 00:00:03,500\nHello world") returns [{id: 1, text: "Hello world", startMs: 1000, endMs: 3500, ...}]
  </behavior>
  <action>
    Create `remotion/src/parseSubtitles.ts` with:

    ```typescript
    export interface SubtitleSegment {
      id: number;
      text: string;
      startFrame: number;
      endFrame: number;
      startMs: number;
      endMs: number;
    }

    export function parseSrt(srtText: string, fps: number = 30): SubtitleSegment[]
    ```

    Implementation:
    1. Split SRT text by blank lines into subtitle blocks
    2. For each block: parse index line, timestamp line (`HH:MM:SS,mmm --> HH:MM:SS,mmm`), and text lines
    3. Convert timestamps to milliseconds via regex: `/(\d{2}):(\d{2}):(\d{2}),(\d{3})/`
    4. Convert ms to frame numbers: `Math.round(ms / 1000 * fps)`
    5. Join multi-line text with space, trim whitespace
    6. Return SubtitleSegment array

    Also create `remotion/src/parseSubtitles.test.ts` with basic assertions for the parser.
  </action>
  <verify>
    <automated>cd remotion && npx tsx src/parseSubtitles.test.ts</automated>
  </verify>
  <done>parseSrt() correctly converts SRT text to SubtitleSegment array with accurate frame timing. Empty input returns [].</done>
