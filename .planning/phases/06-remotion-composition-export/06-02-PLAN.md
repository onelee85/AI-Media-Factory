# Remotion SSR API pattern (from docs):
# 1. bundle({ entryPoint }) -> bundleLocation
# 2. selectComposition({ serveUrl, id, inputProps }) -> composition
# 3. renderMedia({ composition, serveUrl, codec: 'h264', outputLocation, inputProps })
#
# The VideoComposition component (from Plan 01) expects these input props:
#   audioSrc: string       - path to audio file
#   subtitleText: string   - raw SRT text content
#   backgroundImages: string[] - image URLs/paths
#   titleText?: string
#   fps?: number (default 30)
#   width?: number (default 1920)
#   height?: number (default 1080)
#
# FFmpeg encoding options from existing ffmpeg_utils.py:
#   vcodec: libx264, acodec: aac, video_bitrate: 5M, audio_bitrate: 192k
#   pix_fmt: yuv420p, preset: fast, movflags: +faststart
#   These match Remotion's defaults for h264 codec
</context>

  <action>
    Create `remotion/src/render.mjs` — a Node.js ESM script that uses Remotion's SSR APIs to render the VideoComposition to MP4.

    The script should:
    1. Parse CLI arguments: `--props <json-file>` and `--output <path>`
       - Use process.argv parsing (no external CLI library needed)
       - Read props JSON file with `fs.readFileSync`
    2. Bundle the Remotion project:
       ```javascript
       import { bundle } from '@remotion/bundler';
       const bundleLocation = await bundle({
         entryPoint: path.resolve('./src/index.ts'),
         webpackOverride: (config) => config,
       });
       ```
    3. Select the composition:
       ```javascript
       import { selectComposition } from '@remotion/renderer';
       const composition = await selectComposition({
         serveUrl: bundleLocation,
         id: 'VideoComposition',
         inputProps,
       });
       ```
    4. Render to MP4:
       ```javascript
       import { renderMedia } from '@remotion/renderer';
       await renderMedia({
         composition,
         serveUrl: bundleLocation,
         codec: 'h264',
         outputLocation: outputPath,
         inputProps,
       });
       ```
    5. Use `crf: 18` for high quality, `preset: 'fast'` for reasonable speed
    6. Add `--concurrency` option (default: 50% of CPUs via `os.cpus().length / 2`)
    7. Print progress to stdout (use Remotion's `onProgress` callback)
    8. Exit with code 0 on success, code 1 on error (stderr message)
    9. Handle errors gracefully — catch and print, then process.exit(1)

    Also handle the case where audio/image paths in props need to be absolute or resolved relative to the storage root. The render script should resolve relative paths against the project root.

    File format: ESM (.mjs) since Remotion's bundler expects ESM imports.
  </action>
